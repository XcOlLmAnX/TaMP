Подробное описание

Описание текстового протокола TaMP и вспомогательные функции формирования ответов.

Протокол основан на текстовых строках в кодировке UTF-8, разделённых '\n'. Каждый запрос имеет вид: команда&аргумент1&аргумент2…\n. Каждый ответ — команда±[&данные]\n, где '+' означает успех, '-' — ошибку.

◆ CommandType

enum class CommandType
strong

Типы команд, поддерживаемых сервером TaMP.

Элементы перечислений
AUTH 

Авторизация: auth&{login}&{md5hash}.

REG 

Регистрация: reg&{login}&{name}&{md5hash}&{email}.

STAT 

Получение статистики (зарезервировано, не реализовано).

CHECK 

Проверка данных (зарезервировано, не реализовано).

RESET 

Запрос кода сброса пароля: reset&{login}.

CONFIRM 

Подтверждение кода: confirm&{login}&{code}.

NEWPASS 

Установка нового пароля: newpass&{login}&{md5hash}.

UNKNOWN 

Неизвестная или некорректно сформированная команда.

◆ parseRequest()

Request parseRequest ( const std::string & raw)

Разбирает сырую строку клиентского запроса в структуру Request.

Разделитель аргументов — символ '&'. Завершающие '\r' и '\n' обрезаются. Если команда неизвестна или количество аргументов не совпадает с ожидаемым, возвращается Request с type == CommandType::UNKNOWN.

Аргументы
rawСырая строка запроса, полученная от клиента.
Возвращает
Заполненная структура Request.
