Подробное описание

Диспетчер запросов: маршрутизирует команды к специализированным обработчикам.

Принимает разобранный Request и возвращает строку ответа в формате протокола TaMP. Взаимодействует с DbHandler для операций с базой данных и со SmtpClient для отправки email при сбросе пароля.

Временное хранилище кодов сброса (resetCodes) живёт в памяти процесса — коды не персистентны и сбрасываются при перезапуске сервера.

◆ handleReset()

string ServerLogic::handleReset ( const std::vector< std::string > & args)
private

Обрабатывает команду RESET: генерирует код и отправляет его на email.

Ищет email по логину, генерирует 6-значный код и отправляет письмо.

Генерирует 6-значный случайный код (mt19937), сохраняет его в resetCodes и отправляет через SmtpClient::sendPasswordResetCode(). Email маскируется перед отправкой ответа клиенту.

Аргументы
argsАргументы: {login}. При неверном размере или отсутствии логина — ошибка.
Возвращает
Строка ответа с замаскированным email (makeResetOk) или ошибка (makeResetFail).

Код хранится в resetCodes[login] до подтверждения. Email маскируется: все символы до '@' кроме последних двух заменяются на '*****'.
