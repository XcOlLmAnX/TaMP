Подробное описание

Управляет жизненным циклом серверного TCP-сокета и последовательной обработкой клиентов.

Инициализирует QTcpServer, привязывает к порту и переводит в режим прослушивания. Метод run() блокирует вызывающий поток в бесконечном цикле, принимая по одному клиенту за раз (без многопоточности).

Заметки
Однопоточная архитектура: пока один клиент обслуживается, следующий ждёт в очереди.
Исключения
std::runtime_errorпри ошибке listen().

◆ run()

void ServerInterface::run ( )

Запускает бесконечный цикл waitForNewConnection() → handleClient() → …

Для каждого принятого соединения выводит IP клиента в stdout, обрабатывает все его запросы синхронно, затем закрывает сокет.
