Подробное описание

Мастер восстановления пароля из трёх шагов: ввод логина → код → новый пароль.

Шаги переключаются через QStackedWidget:

  • Шаг 1: пользователь вводит логин, сервер отправляет 6-значный код на email.
  • Шаг 2: пользователь вводит полученный код подтверждения.
  • Шаг 3: пользователь задаёт и подтверждает новый пароль.

Все сетевые запросы идут через TcpClient. Ответы фильтруются по префиксам reset, confirm, newpass.

◆ onMessageReceived

void ResetPasswordWindow::onMessageReceived ( const QString & message)
privateslot

Обрабатывает ответы сервера и управляет переключением шагов.

Маршрутизирует ответы сервера по префиксам и обновляет UI или переключает шаг.

Обрабатываемые префиксы:

  • reset+ — переход на шаг 2, отображение замаскированного email;
  • reset- — ошибка: логин не найден;
  • confirm+ — переход на шаг 3;
  • confirm- — ошибка: неверный код;
  • newpass+ — успех, сигнал goToLogin();
  • newpass- — ошибка при сохранении пароля.
Аргументы
messageСтрока ответа от TcpClient::messageReceived().
messageСтрока ответа от TcpClient::messageReceived().

◆ onSetPasswordClicked

void ResetPasswordWindow::onSetPasswordClicked ( )
privateslot

Проверяет совпадение паролей, их сложность и отправляет MD5-хэш серверу.

Валидирует совпадение и сложность паролей, затем отправляет newpass&{login}&{md5}\n.

Формат запроса: newpass&{login}&{md5hash}\n. Требования к паролю: ≥ 6 символов, хотя бы одна цифра.
